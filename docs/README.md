# 文档目录

本目录用于存放当前项目的补充说明文档，重点覆盖整体架构、技术栈与接口设计，方便开发、联调和后续扩展。

## 文档列表

- `architecture.md`
  - 说明系统整体结构、核心数据流、模块职责与关键约束。
- `technology.md`
  - 说明技术选型、依赖职责、运行环境与开发命令。
- `api.md`
  - 说明接口清单、请求响应结构、错误码与联调示例。
- `deployment.md`
  - 说明环境变量、数据库初始化与 Vercel / 其他平台部署建议。
- `database.md`
  - 说明 Prisma 模型、Supabase 连接、建表方式与数据库排障思路。

## 阅读建议

如果你是第一次接手这个项目，推荐按照下面顺序阅读：

1. 先看 `architecture.md`，了解系统全貌与核心闭环。
2. 再看 `technology.md`，了解项目为什么这样选型以及本地如何运行。
3. 再看 `api.md`，方便直接联调接口或对接前端页面。
4. 再看 `deployment.md`，用于本地验证、上线前检查和部署排错。
5. 最后看 `database.md`，用于理解数据模型、建表方式和数据库相关问题排查。
